Data structure choice:
- The data set is implemented as:
  - A simple fixed-size array of STUDENT structs (ID, Age).
  - A small integer array ageCount[13] that counts how many students have each age from 18 to 30.
- This choice is very simple to implement and works well because:
  - The maximum number of students (3000) is known in advance.
  - Insertions and deletions are not frequent.
  - The age range is very small and fixed, which makes maxAgeGap easy to compute.

Worst-case time complexity (n = number of students, A = 13 possible ages):

- createDataSet:
  - Allocates the main array and initializes ageCount to zero (loop over A = 13).
  - Worst-case: O(1)

- destroyDataSet:
  - Frees the array and the DATASET structure.
  - Worst-case: O(1)

- insertion:
  - Appends at end of array and increments ageCount[age - MIN_AGE]. No shifting.
  - Worst-case: O(1)

- searchAge:
  - Linear scan of the entire array; prints every student with the given age.
  - Worst-case: O(n)

- deletion:
  - Linear scan; for each match, overwrite with last element and decrement count.
  - Worst-case: O(n)

- maxAgeGap:
  - Scans ageCount[] (size A = 13) to find smallest and largest age present.
  - Worst-case: O(A) = O(1)

Justification and alternatives:

- Array of structs:
  - Very easy to implement and understand.
  - Works well when the maximum size is known and moderate (3000 students).
- Linked list:
  - Would give similar O(n) time for searchAge and deletion, but adds pointer overhead and more complex code.
  - Does not improve the frequent operations in this application.
- Hash table keyed by age:
  - The age range is extremely small, so a full hash table is unnecessary.
  - The simple ageCount[] array already gives O(1) maxAgeGap and helps track ages.
- Therefore, a plain array plus the small ageCount[] array is an appropriate and simple choice for Application 1.
